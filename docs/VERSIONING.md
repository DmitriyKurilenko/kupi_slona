# Версионирование

## Формат версии
`MAJOR.MINOR.PATCH` (SemVer)

## Текущая версия

Единый источник истины — файл [`/VERSION.md`](../VERSION.md) в корне репозитория.
Это единственное место, где значение нужно править при bump-е.

> **Примечание:** `CHANGELOG.md` пока не создан. Для истории изменений используется [`RELEASE_NOTES.md`](RELEASE_NOTES.md) (пользовательские) и [`DEV_LOG.md`](DEV_LOG.md) (технические).

## Bump-чеклист

При каждом релизе:
1. Обновить `/VERSION.md` (одна строка, без префиксов).
2. Добавить новую секцию сверху в [`/docs/RELEASE_NOTES.md`](RELEASE_NOTES.md) (по-русски, без технических деталей).
3. Добавить запись в [`/docs/DEV_LOG.md`](DEV_LOG.md).
4. В commit message использовать префикс `X.Y.Z:`.

## Когда что бампать

- **PATCH (`0.1.x`)** — bug fixes, hardening, security hygiene, рефакторинг без видимых пользователю изменений.
- **MINOR (`0.x.0`)** — новая пользовательская функциональность, обратно-совместимые изменения API.
- **MAJOR (`x.0.0`)** — breaking changes (пока проект `0.y.z`, MAJOR не зарезервирован под совместимость — после стабилизации API перейдём на `1.0.0`).

## Формат коммитов
```
X.Y.Z: тип(скоуп): описание

Тело (опционально)
```

### Типы
- `feat` — новая функциональность
- `fix` — исправление бага
- `refactor` — рефакторинг без изменения поведения
- `docs` — документация
- `chore` — сборка, зависимости, конфиг
- `test` — тесты

### Примеры
```
0.1.1: fix(accounts): remove email verification on registration
0.2.0: feat(listings): add search by geo distance
0.1.0: feat: barterri v0.1 MVP — listings, offers, chat
```
